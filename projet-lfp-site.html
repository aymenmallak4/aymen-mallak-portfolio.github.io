Projet LFP — Site internet HTML • PHP • CSS

Création du site internet de la LFP permettant aux internautes de visualiser les clubs, les rencontres, les scores et de déposer des commentaires.

Date limite : 08/05/2025 BTS SIO SLAM — 1ʳᵉ année Projet académique
Contexte du projet

La LFP (Ligue de Football Professionnel) souhaitait développer son site internet. Les internautes peuvent visualiser tous les clubs, les rencontres, les scores et inscrire leurs commentaires sur les rencontres. Ce projet constitue la Phase 2 du projet LFP, faisant suite à l'application C# (Phase 1).

commentaire idcom, datecom, libelle, idjournee, numrenc, idutil
utilisateur idutil, pseudoutil, mdputil
Fonctionnalités — User Stories SCRUM
1 Classement des clubs

Visualiser le classement ordonné du 1er au dernier : points cumulés, buts marqués, buts encaissés et différence de buts.

2 Affichage des informations d'un club

Sélectionner un club dans une liste de liens hypertextes pour visualiser ses informations.

3 Résultats d'une journée

Choisir une journée dans une liste déroulante pour afficher les résultats des rencontres.

4 Création d'un compte

Créer un compte avec un pseudo (unicité vérifiée) et un mot de passe.

5 Authentification

S'authentifier avec son pseudo et son mot de passe.

6 Commentaires sur une rencontre

En tant qu'internaute authentifié : choisir une journée, sélectionner une rencontre via lien hypertexte, afficher les commentaires et en déposer un nouveau.

Outils et normes de développement
  • Développement en HTML, CSS, PHP avec bibliothèques éventuelles (Bootstrap, …)
  • Indentation, commentaires, convention de nommage
  • Séparation des couches : HTML, CSS, JavaScript, PHP, dossiers include
  • Mise en place de classes, fonctions ou procédures
  • Accès BDD via l'extension PDO pour MySQL
  • Chaîne de connexion unique dans un fichier .inc.php
Hébergement du site

Le site est installé sur un hébergeur distant gratuit infinityfree.com. Étapes : création d'un compte client, création de la base de données et import du script SQL, upload des fichiers (FileManager ou FTP Filezilla), tests.
